Question 1

Which HTML5 semantic element should be used for the main content of a document?

Your answer:
Correct answer: B - <main>
Explanation:

The <main> element represents the dominant content of the document. There should only be one <main> element per page, and it should not be nested within <article>, <aside>, <footer>, <header>, or <nav> elements. This helps with accessibility and SEO.

Question 2

Which color scheme represents good contrast for accessibility?

Your answer:
Correct answer: Option B
Explanation:

WCAG (Web Content Accessibility Guidelines) requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. Dark text on a light background (or vice versa) provides the best readability and accessibility for users with visual impairments.

Question 3 Reviewed

List and briefly explain three best practices for writing clean, maintainable CSS code.

Your answer:

Expected answer:

1) Use meaningful class names: Choose descriptive names that explain the purpose, not the appearance. 2) Follow a consistent naming convention: Use BEM, SMACSS, or another methodology consistently. 3) Organize code logically: Group related styles together and use comments to separate sections. 4) Avoid excessive specificity: Keep selectors simple to make styles easier to override. 5) Use CSS variables: Define reusable values for colors, spacing, etc.

Explanation:

Clean, maintainable CSS is crucial for long-term project success. It makes collaboration easier, reduces bugs, and speeds up development. Following established conventions and best practices helps create a codebase that's easy to understand and modify.

Question 4

What is the correct way to link an external CSS file in HTML?

Your answer:
Correct answer: B - <link rel="stylesheet" href="styles.css">
Explanation:

The <link> element with rel="stylesheet" is the standard way to include external CSS files. It should be placed in the <head> section of your HTML document. The href attribute specifies the path to the CSS file.

Question 5

Which CSS property controls the stacking order of positioned elements?

Your answer:
Correct answer: B - z-index
Explanation:

The z-index property specifies the stack order of positioned elements (elements with position: relative, absolute, fixed, or sticky). Elements with higher z-index values appear in front of elements with lower values. The default value is auto (0).

Question 6 Reviewed

Describe a scenario where you would use CSS Grid instead of Flexbox, and explain why.

Your answer:

Expected answer:

CSS Grid is ideal for creating complex two-dimensional layouts, such as a full page layout with header, sidebar, main content, and footer. For example, a dashboard with multiple widgets arranged in rows and columns would benefit from Grid because you can control both horizontal and vertical alignment simultaneously. Flexbox is better for one-dimensional layouts (either row or column), while Grid excels at managing both dimensions at once.

Explanation:

While Flexbox and Grid can sometimes be used interchangeably, each has its strengths. Grid is designed for two-dimensional layouts where you need precise control over both rows and columns. Flexbox is better for one-dimensional layouts and when you want items to flexibly fill available space. Often, the best approach is to use Grid for the overall page structure and Flexbox for components within that structure.
